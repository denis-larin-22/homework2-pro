1. false && true || true - Ответ: true. Логический оператор && имеет больший приоритет и по этому выполняеться первым, возвращает первое false, далее оператор || при сравнении (false || true) возвращает true (так как один из аргументов даёт true);
  2. false && (true || true); - Ответ: false. Из-за скобок приоритет смещается на оператор || (или), он выполняеться первым, возвращает true (возвращает первое истинное значение). Далее оператор && при сравнении (false && true) возвращает первое ложное значение false.
  3. +'40' + +'2'; Ответ: 42. Унарный плюс в начале каждой из строк делает преобразование строки в число, далее идет сложение.
  4. '2' + 3 ** 2; Ответ: 29. Из-за большей приоритетности сначала выполняется возведение в степень (3 ** 2 == 9), далее оператор бинарный плюс ('2' + 9), так как один из операндов является строкой, второй также преобразовывается в строку и происходит конкатенация.
  5. 3 ** 2 / 3; Ответ: 3. Из-за большего приоритета, сначала выполняется возведение в степень (3 ** 2 == 9), далее происходит деление (9 / 3).
  6. 3 ** (9 / 3); Ответ: 27. Сначала действие в скобках, затем возведение в степень.
  7. '10' + (5 === 6); Ответ: '10false'. Сначала идет строгое сравнение в скобках которое в результате даёт false, далее, так как есть строка ('10'), идёт конкатенация ('10' + false).
  8. '10' - 5 === 6; Ответ: false. Строка '10' при вычитании преобрaзуется в число, выполниться вычитание (10 - 5 == 5), далее выполняется строгое сравнение (5 === 6), которое вернёт false.
  9. undefined + 1; Ответ: NaN. При преобразование undefined даёт NaN, далее любые вычесления с NaN в результате дают NaN.
  10. (+null == false) < 1; Ответ: false. Сначала действие в скобках. +null - преобразование к числу (0), выполняется сравнение с преобразованием к числу где false даст 0 (0==0), получаем true. Далее при сравнении оператором "<" true преобразуется в 1. Конечное сравнение 1 < 1 // false.
  11. (+null == false) < 1 ** 5; Ответ: false. Логика та же что и в предидущем примере, 1 в 5-ой степени будет 1, конечное сравнение всё то же (1 < 1) // false.
  12. var number = 5;
    2 * ++number;
    2 * number--;
    console.log(number);
    Ответ: 5.     2 * ++number, префиксный инкремент увеличивает number на 1, затем идёт умножение на 2, результат выражения 12. 2 * number--, постфиксный декримент у number, возвращает старое значение, затем выполняется умножение на 2, результат выражения 10. Итоговый результат в консоли будет 5 так как несмотря на две операции с умножением, переменная number не переприсваевается, она лишь увеличивается на 1 во второй строке (++number) и будет 6, затем уменшается на 1 в третей строке (number--) и возвращает старое значение 5.
